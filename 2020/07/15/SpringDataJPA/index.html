Spring Data JPA 使用文档

发表于 2020-07-15 | 分类于 Spring Data

说明

Spring Data JPA是Spring基于ORM框架、JPA规范的基础上封装的一套JPA应用框架，提供了包括增删改查等在内的常用功能，且易于扩展。

Spring Data JPA使我们解脱了DAO层的操作，基本上所有CRUD都可以依赖它实现，实际工作中推荐使用Spring Data JPA + ORM（如Hibernate）完成操作，在切换不同的ORM框架时提供极大的便利，也使数据库层操作更加简单，方便解耦。

Spring Data JPA是Spring提供的一套对JPA操作更加高级的封装，是在JPA规范下的专门用来进行数据持久化的解决方案。

第一部分、环境搭建

1.创建Maven工程，导入依赖项坐标

2.配置Spring整合Spring Data JPA的配置文件

在导入依赖后，我们需要配置以下几点：

1.配置数据库连接池，具体使用哪种数据库连接池任意选择；

2.配置实体类管理器工厂；

3.配置事务管理器；

4.指定Repository接口所在的包，Spring Data JPA将对此包下的Repository接口进行动态代理增强。

第二部分、简单CRUD操作

1.定义实体类，配置实体类到数据库表以及实体类属性到数据库字段的映射关系

2.定义Repository接口，继承CrudRepository接口

自定义Repository接口继承CrudRepository接口后，不需要实现其中的任何方法，可以直接调用其中的save、findById、delete等方法。

3.使用自定义Repository接口完成CRUD操作

可以看到，在测试方法中，通过直接调用PersonDao对象继承得到的findById、save、delete方法就可以完成对数据库的查询、更新、删除和增加功能，而不需要实现任何方法。

第三部分、自定义方法

Spring Data JPA按照方法名解析规则如下：

find+全局修饰+By+实体属性名称+限定词+连接词+（其他实体属性）+OrderBy+排序属性+排序方向

例如：

关键词 样例 JPQL样例

Or findByNameOrAge(String name, Integer age) where x.name= ?1 or x.age= ?2

LessThan findByAgeLessThan(Integer age) where x.age < ?1
GreaterThan findByAgeGreaterThan(Integer age) where x.age > ?1

NotLike findByNameNotLike(String name) where x.name not like ?1

第四部分、分页&排序数据查询

1.分页查询

使用分页查询，需要自定义Repository接口继承PagingAndSortingRepository接口，从而使用其中的分页查询方法。

之后，便可构造Pageable对象，使用findAll(Pageable pageable)方法获取分页查询结果。

2.排序查询

同样，自定义Repository接口继承PagingAndSortingRepository接口，从而使用其中的排序查询方法。

第五部分、自定义JPQL查询语句

在按规范自定义方法不满足要求的情况下，Spring Data JPA 还可以使用@Query搭配@Modifying完成自定义JPQL查询。

@Query注解

@Query注解查询适用于所查询的数据无法通过关键字查询得到结果的查询。这种查询可以摆脱像关键字查询那样的约束，将查询直接在相应的接口方法中声明，结构更为清晰。

  1. 索引参数如下所示，索引值从1开始，查询中“?X”个数需要与方法定义的参数个数相一致，并且顺序也要一致。

  1. 命名参数(推荐使用此方式)：可以定义好参数名，赋值时使用@Param(“参数名”),而不用管顺序。

  1. 含有LIKE关键字的查询

    方式1：在占位符上添加”%”，这样在查询方法中就不用添加”%”

方式2：不在占位符上添加”%”,这样就必须在查询方法的参数上添加”%”

方式3：在命名参数上添加”%”

  1. 使用原生SQL进行查询

    当设置nativeQuery=true，即可以使用原生SQL进行查询。

@Modifying注解

  1. 在@Query注解中编写JPQL实现DELETE和UPDATE操作的时候必须加上@Modifying注解，以通知Spring Data 这是一个DELETE或UPDATE操作。

  2. UPDATE或者DELETE操作需要使用事务，此时需要定义Service层，在Service层的方法上添加事务操作。

  3. 注意JPQL不支持INSERT操作。

第七部分、Example查询

Example查询按例查询，是一种用户界面友好的查询技术。 它允许动态创建查询，并且不需要编写包含字段名称的查询。 而且按示例查询不需要使用特定的数据库的查询语言来编写查询语句。

使用方法

1.创建一个Entity实体；

2.传入Example.of()方法构造一个Example对象；

3.使用自定义Repository接口继承了QueryByExampleExecutor得到的findAll(Example<S> example)方法。

优势：
  1. 可以使用动态或者静态的限制去查询；
  2. 在重构你的实体的时候，不用担心影响到已有的查询；
  3. 可以独立地工作在数据查询API之外。
劣势：
  1. 不支持组合查询，比如：firstname = ?0 or (firstname = ?1 and lastname = ?2)；
  2. 只支持字符串的starts/contains/ends/regex匹配，对于非字符串的属性，只支持精确匹配。
原理

从生成的SQL语句我们可以看到，它的判断条件是根据实体的属性来生成查询语句的。如果实体的属性是null，它就会忽略它；如果不是，就会取其值作为匹配条件。
